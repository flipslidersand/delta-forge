# ADR-001: 実装言語に Rust を選ぶ

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

増分計算エンジンはグラフ走査・クロージャ管理・キャッシュ更新が頻発する。
メモリ安全性と型安全性を保ちながら、`Box<dyn Any>` による型消去も必要。

## 決定

Rust を使う。

## 理由

- 所有権モデルにより「ノードが値を持ち、エンジンがグラフを所有する」構造を安全に表現できる
- petgraph が Rust ネイティブで、グラフ走査・トポロジカルソートが標準 API として揃っている
- `Send + Sync` 制約により、Phase 6 で Rayon 並列化する際の安全性をコンパイル時に保証できる

## トレードオフ

- `Box<dyn Any + Send>` の型消去で downcast エラーが実行時になる
- クロージャのライフタイム管理が Python/JS より複雑
