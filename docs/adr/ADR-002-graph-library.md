# ADR-002: グラフ実装に petgraph を使う

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

依存グラフの実装方法として、手書き隣接リスト・petgraph・カスタム DAG ライブラリの選択肢がある。

## 決定

`petgraph` を使う。

## 理由

- `toposort()` と `is_cyclic_directed()` が標準提供されており、Phase 1〜3 の核心機能を自前実装しなくて済む
- `Bfs` / `Dfs` イテレータで dirty 伝播を自然に書ける
- Rust コミュニティで最も実績があり、ドキュメントが充実している

## トレードオフ

- `NodeIndex` が削除操作後に無効化されるため、ノード削除を伴う動的グラフ更新が難しい
  → MVP では削除不要のためスコープ外とする
