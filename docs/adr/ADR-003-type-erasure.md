# ADR-003: ノード値に Box<dyn Any + Send> を使う

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

ノードの値は任意の型（i64・Vec<Row>・String 等）になりうる。
型パラメータをエンジン全体に伝播させると、API が爆発的に複雑になる。

## 決定

ノード値を `Box<dyn Any + Send>` で型消去し、利用側が `downcast_ref::<T>()` で取り出す。

## 理由

- エンジンの struct・メソッドをジェネリクスフリーに保てる
- petgraph の `Graph<Node, ()>` に異なる型のノードを格納できる
- Rust 標準ライブラリの機能だけで実現でき、外部クレートへの依存がない

## トレードオフ

- downcast の型ミスが実行時パニックになる → ノード登録時に型タグを持たせて検出を早める (Phase 2 以降)
- `dyn Any` は `Clone` を実装できないため、値の複製に工夫が必要
